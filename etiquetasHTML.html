ETIQUETAS HTML

Elemento Descripcion Documentacion
<a> Crea un enlace a otras páginas de internet, archivos o ubicaciones dentro de la misma página, direcciones de correo, o cualquier otra URL. a
<abbr> Representa una abreviación o acrónimo abbr
<address> El elemento address indica que el HTML contenido proporciona información de contacto para una persona o personas, o para una organización. address
<area> Sirve para crear las distintas secciones body area
<article> Representa una composición auto-contenida en un documento, página, una aplicación o en el sitio, que se destina a distribuir de forma independiente o reutilizable, por ejemplo, en la indicación. article
<aside> El elemento representa una parte de un documento cuyo contenido solo está indirectamente relacionado con el contenido principal del documento. aside
<audio> El elemento se utiliza para incrustar contenido de sonido en los documentos. Puede contener una o más fuentes de audio, representadas mediante el atributo src o el elemento source: el navegador elegirá la más adecuada. audio
<b> Indica que el texto debe ser representado con una variable bold, o negrita, de la tipografía que se esté usando. b
<bdi> Le dice al algoritmo bidireccional del navegador que trate el texto que contiene de forma aislada del texto que lo rodea. bdi
<bdo> Anula la direccionalidad actual del texto, de modo que el texto que contiene se representa en una dirección diferente. bdo
<base> Especifica la dirección URL base que se utilizará para todas las direcciones URL relativas contenidas dentro de un documento. base
<body> Representa el contenido de un documento HTML. Solo puede haber un elemento body en un documento. body
<blockquote> El elemento blockquote indica que el texto contenido es una cita extensa. blockquote
<br> Representa un salto de linea en el documento. br
<button> Representa un elemento cliqueable de tipo botón que puede ser utilizado en formularios o en cualquier parte de la página que necesite un botón estándar y simple de aplicar. button
<canvas> Utilice el elemento HTML canvas con la API de secuencias de comandos de lienzo o la API de WebGL para dibujar gráficos y animaciones. canvas
<cite> El atributo cite especifica la fuente de la cita. El valor del atributo debería ser un enlace al documento original o fuente. cite
<code> El elemento code muestra su contenido con formato para indicar que el texto es un fragmento corto de código de computadora. code
<data> El elemento vincula una determinada pieza de contenido con una traducción legible por máquina. data
<datalist> .Contiene un conjunto de elementos option que representan las opciones permitidas o recomendadas disponibles para elegir dentro de otros controles. datalist
<del> El elemento del marca las partes de un texto que han sido eliminadas del documento. del
<details> Crea un widget de divulgación en el que la información es visible solo cuando el widget se cambia a un estado "abierto". details
<div> Sirve para crear secciones o agrupar contenidos. div
<dfn> El elemento dfn se utiliza para identificar el término a definir, mientras que la definición del término se coloca fuera del elemento. dfn
<dl> Representa una lista de descripcion. dl
<em> Cuando queremos dar énfasis al lenguaje hablado, acentuamos y prolongamos la pronunciación de ciertas palabras y así alteramos sutilmente el significado de lo que decimos. em
<embed> Incrusta contenido externo en el punto especificado del documento. embed
<fieldset> Se utiliza para agrupar varios controles, así como etiquetas dentro de un formulario web. fieldset
<figure> Representa contenido autónomo, potencialmente con un título opcional, que se especifica mediante el elemento figcaption. figure
<footer> Representa un pie de página para su ancestro más cercano que secciona el contenido o secciona el elemento raíz. footer
<form> Sirve para representar una seccion de un documento que contiene controles interactivos para poder subir informacion. Documentacion
<h1-h6> Los elementos de encabezado implementan seis niveles de encabezado del documento. h1-h6
<head> Contiene lo que no se ve en la pagina web head
<header> Representa un grupo de ayudas introductorias o de navegación. Puede contener algunos elementos de encabezado, así como también un logo, un formulario de búsqueda, un nombre de autor y otros componentes. header
<hr> El elemento hr representa una transición entre párrafos a un tema separado, pero relacionado. hr
<hgroup> Representa un encabezado y contenido relacionado. hgroup
<html> Representa la raiz del documento HTML html
<i> El elemento representa un rango de texto que se separa del texto normal por alguna razón, como texto idiomático, términos técnicos, designaciones taxonómicas, entre otros. i
<iframe> El elemento representa un contexto de navegación anidado, incrustando otra página HTML en la actual. iframe
<input> Se usa para crear controles interactivos para formularios basados en la web con el fin de recibir datos del usuario. input
<ins> El elemento ins marca las partes de un texto que han sido añadidos al documento. ins
<img> Representa una imagen en el documento. img
<kbd> Representa un fragmento de texto en línea que indica la entrada del usuario desde un teclado. kbd
<label> Representa un título para un elemento en una interfaz de usuario. label
<li> Es usado para representar un elemento en una lista. li
<link> Especifica la relación entre el documento actual y un recurso externo. link
<main> Representa el contenido dominante del body de un documento. main
<map> Se utiliza con elementos area para definir un mapa de imagen (un área de enlace en la que se puede hacer clic). map
<menu> Se describe en la especificación HTML como una alternativa semántica a ul, pero los navegadores lo tratan (y lo exponen a través del árbol de accesibilidad) como no diferente de ul. menu
<mark> El elemento mark permite resaltar texto de interés. Piense en esto como si usara un resaltador en un libro para marcar pasajes que encuentre de interés. mark
<meta> Sirve para aportar información sobre el documento. meta
<nav> Representa una seccion de la paguna, la cual tiene el proposito de proporcionar links de navegacion. nav
<noscript> Define una sección de HTML que se insertará si un tipo de secuencia de comandos en la página no es compatible o si las secuencias de comandos están actualmente desactivadas en el navegador. noscript
<object> Representa un recurso externo, que se puede tratar como una imagen, un contexto de navegación anidado o un recurso para ser manejado por un complemento. object
<ol> Representa elementos ordenados en una lista. ol
<output> Es un elemento contenedor en el que un sitio o aplicación puede inyectar los resultados de un cálculo o el resultado de una acción del usuario. output
<p> Este elemento representa un parrafo. p
<picture> contiene cero o más elementos source y un elemento img para ofrecer versiones alternativas de una imagen para diferentes escenarios de pantalla/dispositivo. picture
<pre> Representa texto preformateado. El texto en este elemento típicamente se muestra en una fuente fija, no proporcional, exactamente como es mostrado en el archivo pre
<progress> Muestra un indicador que muestra el progreso de finalización de una tarea, que normalmente se muestra como una barra de progreso. progress
<q> El elemento q indica que el texto encerrado es una cita corta en línea. q
<ruby> Representa pequeñas anotaciones que se representan encima, debajo o al lado del texto base, generalmente utilizadas para mostrar la pronunciación de los caracteres del este asiático. ruby
<s> El elemento s muestra el texto tachado con una línea que lo atraviesa. s
<samp> Se utiliza para encerrar texto que representa la salida de un programa de computadora. samp
<section> Representa una sección independiente genérica de un documento, que no tiene un elemento semántico más específico para representarlo. section
<select> Representa un control que proporciona un menú de opciones. select
<script> Se utiliza para incrustar código o datos ejecutables; esto generalmente se usa para incrustar o hacer referencia al código JavaScript. script
<span> Es un contenedor en línea. Sirve para aplicar estilo al texto o agrupar elementos en línea. span
<style> Es el elemento encargado de indicar la información de estilo. style
<small> El elemento small permite definir la letra chica y los comentarios secundarios de una página, como descargos de responsabilidad, derechos de autor, información sobre licencias y restricciones legales. small
<sub> Permite definir subindices. sub
<sup> Permite definir superindices. sup
<strong> El elemento strong indica que su contenido tiene una gran importancia, seriedad o urgencia. Los navegadores normalmente muestran el contenido en negrita. strong
<table> Representa informacion tabular. Representa datos en dos o mas dimensiones table
<tbody> Encapsula un conjunto de filas de tabla tbody
<template> Es un mecanismo para contener HTML que no se debe representar inmediatamente cuando se carga una página, pero se puede crear una instancia posteriormente durante el tiempo de ejecución mediante JavaScript. template
<textarea> Representa un control de edición de texto sin formato de varias líneas, útil cuando desea permitir que los usuarios ingresen una cantidad considerable de texto de forma libre, por ejemplo, un comentario en una revisión o formulario de comentarios. textarea
<thead> Define un conjunto de filas que definen la cabecera de las columnas de la tabla. thead
<time> Representa un período específico en el tiempo. Puede incluir el atributo de fecha y hora para traducir las fechas a un formato legible por máquina, lo que permite obtener mejores resultados en los motores de búsqueda o funciones personalizadas, como recordatorios. time
<title> Define el titulo del elemtno que se vera en la barra dek navegador. title
<u> Representa un tramo de texto en línea que debe representarse de manera que indique que tiene una anotación no textual. u
<ul> Representa elementos desordenados en una lista. ul
<var> Representa el nombre de una variable en una expresión matemática o un contexto de programación. var
<video> Incrusta un reproductor multimedia que admite la reproducción de video en el documento. video
<wbr> El elemento vacío wbr le indica al navegador donde podría razonablemente insertar un salto de línea para ajustar el contenido que es más grande que la ventana actual del navegador wbr
